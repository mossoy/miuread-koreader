# MiuRead v1.0.1

## 想法窗口关闭与评论定位修复

- 将想法卡片右上角 `×` 从纯文本覆盖层改为 KOReader 原生可点击按钮。
- 保留独立关闭热区作为兼容后备，避免点击 `×` 被卡片翻页逻辑接管。
- 调整章节处理顺序：先按微信读书原始正文注入划线与想法，再转换脚注。
- 修复含脚注书籍中，脚注编号和尾注内容导致后续评论位置逐步偏移的问题。
- 评论定位优先使用划线原文进行文本校准，兼容 HTML 实体、排版空白和隐藏标签。
- 数字位置回退增加 UTF-16 映射，降低含 emoji 或扩展字符书籍中的偏移。
- 下载断点缓存结构升级为 schema 3，旧的未完成断点会自动重建。

> 已下载的旧“划线与想法版”不会自动修复，需要重新下载对应书籍或章节。
