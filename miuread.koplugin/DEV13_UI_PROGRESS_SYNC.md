# MiuRead 0.1.0-dev.13 · UI整理与阅读进度同步

## 本版定位

本版基于 `0.1.0-dev.12`，保留连续想法窗口、局部刷新、30秒阅读时间上报和单实例后台服务；主要调整菜单结构，并重新启用阅读进度同步。

## UI调整

- 首页精简为：我的书架、搜索书籍、下载管理、阅读同步、设置。
- “我的书架”直接进入普通书籍书架。
- 长按书架中的书籍可切换普通书籍/公众号、刷新书架、设置排序和筛选。
- 书籍与章节菜单根据本地下载状态动态显示，不再展示不可用的“打开”入口。
- 下载管理直接显示“纯净版”“划线与想法版”“单章缓存”，不再显示内部 `clean/notes` 名称。
- 设置按下载、阅读与想法、书架显示、性能、存储与缓存、账户、更新与关于分组。
- “评论字体大小”统一改为“想法字体大小”。
- 删除书籍、缓存和账户信息时增加明确确认说明。

## 阅读进度同步

- 默认开启阅读进度同步；升级用户若此前没有该选项，也会自动开启。
- 打开觅阅书籍约1秒后，在后台读取一次微信读书云端位置。
- 本机与云端相差不超过2%：自动视为一致。
- 本机位置明显领先：自动继续使用本机位置。
- 云端位置明显领先：询问“跳到云端位置 / 继续使用本机位置 / 取消”。
- 用户选择本机位置、云端位置或取消时，阅读时间上报均不会暂停。
- 阅读时间仍由原有30秒兼容链路上报，未增加逐页网络请求。

## 同步状态

普通状态页只显示：

- 阅读时间状态
- 阅读进度状态
- 当前本机位置
- 已获取时的云端位置
- 上次同步时间

HTTP状态、响应长度、服务PID、原始错误等信息移动到“高级 → 查看详细同步信息”。成功恢复后会清除普通状态页中的旧错误。
